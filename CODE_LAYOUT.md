# Code layout — habit-coach-web

Conventions for **this** repo so we don’t rely on a late refactor. Pages stay thin; logic and shared bits have a clear home.

## Principles

- **Thin entrypoints** — Route files compose UI; avoid large blocks of logic, constants, or raw GraphQL strings in `app/**/page.tsx`.
- **No magic strings** — Values used more than once → `constants/` or a named export.
- **Pure helpers** — Small functions with no React → `lib/` with a **specific** filename (e.g. `lib/format-date.ts`), not one giant `utils.ts`.
- **Colocation** — If only one feature uses a helper, keep it next to that feature until reuse is real.
- **Path aliases** — Use `@/` from the start if configured (`@/components/...`, `@/lib/...`).

## Directories

| Area                           | Purpose                                                                                       |
| ------------------------------ | --------------------------------------------------------------------------------------------- |
| `app/`                         | Routing: `page.tsx`, `layout.tsx`, `loading.tsx`, `error.tsx`, route handlers. Minimal logic. |
| `components/ui/`               | Reusable primitives (buttons, inputs, layout shells).                                         |
| `components/<feature>/`        | Feature UI (e.g. `components/habits/`).                                                       |
| `lib/`                         | Apollo client, auth/token helpers, env-safe config, non-UI glue.                              |
| `hooks/`                       | Shared hooks (`useAuth`, `useHabits`). One-off hooks may live next to the feature.            |
| `constants/`                   | Fixed values, paths, defaults — not duplicated across JSX.                                    |
| `types/`                       | Shared TS types not generated by GraphQL Codegen.                                             |
| `schemas/`                     | Shared Zod schemas when the same rules appear in multiple places.                             |
| `graphql/` _or_ `lib/graphql/` | Operation documents + codegen output — **pick one** place and keep it consistent.             |

## Rules of thumb

- Page or layout **> ~100–150 lines** → extract components and hooks.
- New operation → typed document + Apollo; don’t scatter duplicate query strings.
- **Tests:** colocate `*.test.tsx` next to the component or use one `__tests__/` convention per area.

## Cross-repo

The GraphQL contract is defined in **`habit-coach-api`**. Coordinate breaking changes with the frontend.
